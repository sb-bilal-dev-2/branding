Гибридный поиск Postgres + Qdrant: когда BM25 побеждает векторный

30 января 2025 · 5 минут · поиск, RAG, Postgres
Что вы получите за 5 минут
  • Класс запросов, где векторный поиск проигрывает с разгромным счётом
  • Формула reciprocal rank fusion в одну строку и почему она работает
  • Реальные числа MRR из небольшого эвал-сета: BM25 vs vector vs hybrid

Где BM25 побеждает векторный поиск

Векторный поиск отлично улавливает смысл, но плохо — точную форму. Запросы, где BM25 неожиданно выигрывает:

  • Номер документа — «договор ARA-2026-014». Эмбеддинги превращают номер в нечто среднее по всем номерам.
  • Имя собственное — «ООО Бином Сервис». Часто проседает на близких по форме именах.
  • Точные цитаты — «дата подписания 14.03.2026». Векторы дают близкое по контексту, не по содержанию.
  • Жаргон / редкая терминология — то, чего мало в обучающем корпусе эмбеддера.

Reciprocal Rank Fusion в одну формулу

RRF — простой способ объединить два разных ранкинга без необходимости их нормализовать. Для каждого документа считаем:

score(d) = 1/(k + rank_BM25(d)) + 1/(k + rank_vector(d))

где k обычно 60. Что это даёт:

  • Документ, попавший в топ обоих ранкингов, поднимается вверх.
  • Документ из топа только одного ранкинга — остаётся в результатах, но ниже.
  • Никаких настраиваемых весов — баланс задаётся самим k.

Реальные числа

Скрипт eval_search.py в проекте ARA прогоняет 10 запросов против 10-документного корпуса (узбекский, русский, английский смешанные). Результаты на этом сэте:

  • Keyword (BM25): MRR@5 = 1.000, Recall@5 = 1.000
  • Vector: MRR@5 = 0.173, Recall@5 = 0.600
  • Hybrid (RRF): MRR@5 = 0.950, Recall@5 = 1.000

На этом маленьком фиксированном сэте BM25 выигрывает за счёт того, что многие запросы содержат точные формы (номера, имена). Vector проседает. Hybrid сохраняет выигрыш BM25 и при этом не теряет на смысловых запросах. Это причина не выбирать одно из двух.

Где это в Postgres

BM25-подобный ранкинг в Postgres делается через ts_rank поверх tsvector. Один GIN-индекс — и поиск работает на миллионах строк за миллисекунды. В ARA tsvector создан с конфигурацией simple — без стемминга, чтобы одинаково обращаться с узбекским, русским и английским контентом.

Что теперь можно сделать
  • Добавьте tsvector + GIN-индекс рядом с вашими эмбеддингами.
  • Реализуйте RRF — это 10 строк кода на Python.
  • Соберите эвал-сет из 20 ваших реальных запросов и замерьте все три режима.
  • Посмотрите переключатель «Hybrid / Vector / Keyword» в демо ARA.
